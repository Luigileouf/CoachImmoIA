# Ecrans Prioritaires Web

## Lot P0

Les premiers ecrans a maquetter puis integrer pour la plateforme web sont :

## 1. Dashboard Acheteur / Vendeur

Role :

- donner une vue immediate du dossier actif
- montrer progression, alertes et prochaines actions

Zones clefs :

- header produit
- cards KPI
- timeline projet
- focus du jour
- panneau lateral coach / IA

## 2. Biens

Role :

- consulter, filtrer et comparer les biens ou comparables

Zones clefs :

- filtres
- liste principale
- carte detail ou panneau detail
- tags de priorite

## 3. Assistant IA Workspace

Role :

- transformer le chat en veritable espace de travail

Zones clefs :

- conversation
- prompts guidants
- sources dossier
- resume dynamique
- CTA vers coach humain

## 4. Projet

Role :

- piloter les etapes et les dependances du dossier

Zones clefs :

- roadmap
- checklist
- bloc risque
- actions prioritaires

## 5. Documents

Role :

- suivre l'etat documentaire et preparer le futur RAG

Zones clefs :

- liste ou tableau documents
- statuts
- filtres
- documents manquants
- actions de relance

## 6. Profil / Parametres

Role :

- regler le compte, l'accompagnement et les preferences IA

Zones clefs :

- identite dossier
- mode accompagnement
- securite
- notifications

## Recommandation de maquette

Pour aller vite, la passe Figma desktop doit produire :

- `01 Dashboard`
- `02 Listings`
- `03 Assistant Workspace`
- `04 Project Workspace`
- `05 Documents`
- `06 Profile`

## Recommandation de build

Ordre de developpement conseille :

1. `Dashboard`
2. `Assistant Workspace`
3. `Projet`
4. `Documents`
5. `Biens`
6. `Profil`
